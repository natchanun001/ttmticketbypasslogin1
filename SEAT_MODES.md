# โหมดการเลือกที่นั่ง (Seat Selection Modes)

บอท TTM Ticket Bypass รองรับการเลือกที่นั่งหลายรูปแบบ เพื่อให้เหมาะสมกับความต้องการและลดการแย่งที่นั่งกันเองเมื่อรันหลายจอพร้อมกัน

## วิธีตั้งค่า
แก้ไขในไฟล์ `.env` ที่หัวข้อ `TTM_SEAT_MODE`:
```env
# สำหรับจอเดียว
TTM_SEAT_MODE=FRONT_LEFT

# สำหรับหลายจอ (คั่นด้วยคอมม่า)
TTM_SEAT_MODE=FRONT_LEFT,RANDOM,BACK_RIGHT
```

---

## รายละเอียดโหมดต่างๆ

| ชื่อโหมด | คำอธิบาย | พฤติกรรม |
| :--- | :--- | :--- |
| **`FRONT_LEFT`** (Default) | แถวหน้า-ซ้ายสุด | เริ่มจากแถวหน้าสุด (A, B, C...) และเลือกกลุ่มที่นั่งติดกันที่อยู่ฝั่งซ้ายสุดของแถวนั้น |
| **`FRONT_RIGHT`** | แถวหน้า-ขวาสุด | เริ่มจากแถวหน้าสุด แต่เลือกกลุ่มที่นั่งติดกันที่อยู่ฝั่งขวาสุดของแถวนั้น |
| **`BACK_LEFT`** | แถวหลัง-ซ้ายสุด | เริ่มจากแถวหลังสุด (Z, Y, X...) และเลือกกลุ่มที่นั่งติดกันจากฝั่งซ้าย |
| **`BACK_RIGHT`** | แถวหลัง-ขวาสุด | เริ่มจากแถวหลังสุด และเลือกกลุ่มที่นั่งติดกันจากฝั่งขวา |
| **`CENTER`** | แถวกึ่งกลาง | เริ่มจากแถวหน้าสุด แต่เลือกกลุ่มที่นั่งติดกันที่อยู่ **"ใกล้จุดกึ่งกลางของแถว"** มากที่สุด |
| **`RANDOM`** | สุ่มที่นั่งติดกัน | สุ่มเลือกแถว และสุ่มเลือกกลุ่มที่นั่งติดกันในแถวนั้นๆ |
| **`ANY_RANDOM`** | สุ่มที่นั่ง (ไม่ติดกัน) | **ไม่สนใจความต่อเนื่อง** บอทจะหยิบที่นั่งว่างตรงไหนก็ได้ในโซนมาให้ครบจำนวน (เหมาะสำหรับบัตรหายากมาก) |

---

## ข้อแนะนำสำหรับการรันหลายจอ
หากคุณรันบอทพร้อมกัน 3 จอ แนะนำให้ใช้โหมดผสมกันเพื่อไม่ให้บอทวิ่งไปแย่งที่นั่งจุดเดียวกัน เช่น:
`TTM_SEAT_MODE=FRONT_LEFT,FRONT_RIGHT,CENTER`

การทำแบบนี้จะทำให้:
*   บอทตัวที่ 1 วิ่งไปแถวหน้าฝั่งซ้าย
*   บอทตัวที่ 2 วิ่งไปแถวหน้าฝั่งขวา
*   บอทตัวที่ 3 วิ่งไปแถวหน้าตรงกลาง
**โอกาสสำเร็จจะสูงขึ้นมากเพราะบอทช่วยกันกระจายความเสี่ยง**
